# Steiner Tree Solver (L1 Manhattan Metric)

## 📋 Постановка задачи

Реализовать программу для построения **дерева Штейнера минимальной длины (SMT)** на плоскости с манхэттенской метрикой (L1). Программа должна:

- Строить дерево, используя **базовый алгоритм 1IS** (итеративное добавление одной точки Штейнера из сетки Ханнана).
- Предложить **модификацию алгоритма**, улучшающую производительность.
- Сравнить базовый и модифицированный алгоритмы по времени выполнения и качеству решения.

**Метрика расстояния:**

$$
    L = | x_1 - x_2 | + | y_1 - y_2 |
$$

## 🧠 Алгоритмы

### 1. Базовый алгоритм (1IS)

На каждой итерации:
1. Строится минимальное остовное дерево (MST) текущего множества точек.
2. Перебираются **все кандидаты сетки Ханнана** (все комбинации x- и y-координат терминалов, не совпадающие с существующими точками).
3. Выбирается кандидат, добавление которого **максимально сокращает длину MST**.
4. Точка добавляется как точка Штейнера (`type: "s"`).
5. Процесс повторяется, пока есть улучшение.

### 2. Модифицированный алгоритм

Отличие от базового — **фильтрация кандидатов** перед проверкой:
- На каждой итерации строятся **прямоугольники, ограничивающие каждое ребро текущего MST**.
- Рассматриваются только те кандидаты сетки Ханнана, которые попадают **хотя бы в один такой прямоугольник**.

Это позволяет **существенно сократить число проверяемых кандидатов**, сохраняя качество решения близким к оптимальному.

## ⚙️ Требования и запуск

### Системные требования
- Python 3.8+
- Библиотеки: `matplotlib` (опционально, для визуализации)

### Установка
```bash
git clone https://github.com/GauchKirill/SteinerTree
cd SteinerTree
pip install -r requirements.txt  # если нужна визуализация matplotlib
```

## Запуск программы
```bash
# Базовый алгоритм
python steiner.py <test_name>.json

# Модифицированный алгоритм
python steiner.py -m <test_name>.json

# С визуализацией (сохраняется в PNG)
python steiner.py -m -v <test_name>.json
```
### Выходные файлы

  <test_name>_out.json — результат в формате JSON (точки и рёбра).

  <test_name>_out.png — визуализация дерева (при флаге -v).

### Пояснения:
| Столбец | Пояснение |
|---------|-----------|
| **Test** | Имя тестового файла (входные данные) |
| **Term** | Количество терминальных точек (type: "t") во входном файле |
| **Init MST** | Длина минимального остовного дерева (MST), построенного только на терминалах, без точек Штейнера |
| **Basic Final** | Финальная длина дерева Штейнера, полученная базовым алгоритмом |
| **Basic Time** | Время выполнения базового алгоритма в секундах |
| **St (B)** | Количество точек Штейнера в решении, полученном базовым алгоритмом |
| **Mod Final** | Финальная длина дерева Штейнера, полученная модифицированным алгоритмом |
| **Mod Time** | Время выполнения модифицированного алгоритма в секундах |
| **St (M)** | Количество точек Штейнера в решении, полученном модифицированным алгоритмом |
| **Speedup** | Ускорение модифицированного алгоритма относительно базового. Вычисляется как `Basic Time / Mod Time`. Значение > 1 означает, что модифицированный алгоритм быстрее |
| **Impr %** | Процент улучшения финальной длины относительно начальной MST. Вычисляется как `(Init MST - Basic Final) / Init MST * 100%`. Показывает, насколько дерево Штейнера короче обычного остовного дерева |

### 📈 Анализ результатов

#### Сравнение времени выполнения

Модифицированный алгоритм демонстрирует **стабильное ускорение** относительно базового:

- **Среднее ускорение:** 3.21x
- **Максимальное ускорение:** 5.06x (test6.json, 50 терминалов)
- **Минимальное ускорение:** 1.06x (test16.json, 8 терминалов)

Ускорение **растёт с увеличением числа терминалов**, что хорошо видно при сравнении тестов:
- Тесты с 7-20 терминалами: ускорение **1.06x – 2.47x**
- Тесты с 32-75 терминалами: ускорение **3.49x – 5.06x**

Это объясняется тем, что базовый алгоритм перебирает все O(N²) кандидатов сетки Ханнана, в то время как модифицированный алгоритм фильтрует большинство из них, оставляя только точки в прямоугольниках текущих рёбер MST. При малом числе терминалов разница в количестве проверяемых кандидатов невелика, но с ростом N она становится критической.

#### Сравнение качества решения

- В **19 из 20 тестов** оба алгоритма нашли деревья **одинаковой длины**.
- В одном тесте (test4.json) модифицированный алгоритм дал длину **505** против **504** у базового (разница **0.2%**).

Это подтверждает, что фильтрация кандидатов по bounding box рёбер **практически не ухудшает** качество решения, но даёт **кратное ускорение**. Потеря качества в одном тесте связана с тем, что оптимальная точка Штейнера могла находиться чуть за пределами bounding box текущего ребра на момент её добавления.

#### Улучшение относительно MST

Добавление точек Штейнера позволило сократить длину соединений:

- **Среднее улучшение:** 11.5%
- **Максимальное улучшение:** 15.9% (test11.json)
- **Минимальное улучшение:** 4.9% (test13.json)

Процент улучшения зависит от взаимного расположения терминалов. Наибольший выигрыш достигается, когда терминалы образуют "L"-образные или "T"-образные структуры, где добавление точки Штейнера в точке излома позволяет существенно сократить суммарную длину.

#### Количество точек Штейнера

Оба алгоритма добавляют примерно одинаковое число точек Штейнера.

В тесте test17.json модифицированный алгоритм добавил **6 точек** против **5** у базового, но при этом нашёл дерево **меньшей длины** (75 против 76). Это показывает, что иногда большее число точек Штейнера может дать лучшее решение.

#### Общие выводы

1. **Производительность:** Модифицированный алгоритм работает в **3-5 раз быстрее** на больших тестах.
2. **Качество:** Длина дерева совпадает в **95% тестов**, а в оставшихся отличается не более чем на **0.2%**.
3. **Масштабируемость:** Ускорение растёт с увеличением числа терминалов, что делает модифицированный алгоритм особенно полезным для практических задач.
4. **Рекомендация:** Для задач с числом терминалов **> 20** следует использовать модифицированный алгоритм, так как он даёт существенный выигрыш по времени при минимальных потерях в качестве.
